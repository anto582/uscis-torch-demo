Token lifecycle & secret protection

⟳Token lifecycle

  1. POST to the USCIS token endpoint (client-credentials) with the env Client ID/Secret.
  2. Receive an access_token (~30 min); cache it for reuse.
  3. Every Case Status call sends Authorization: Bearer <token>.
  4. On a 401 → refresh the token and retry once, automatically.

🔒Protecting Client ID / Secret

  • Stored as environment variables, loaded by systemd from a protected EnvironmentFile. Never in code, never in git.
  • Never logged: secret & full token redacted; Client ID masked on every line.
  • Token & secret stay server-side — the browser never receives them.

Server security

Secrets in .env · encryption · MySQL 8

🔒Secrets — nothing public

  • All keys live in .env, loaded by systemd EnvironmentFile. .env is gitignored — never committed, never in the browser.
  • Fail-closed boot: SECRET_KEY · TP_JWT_SECRET · INTERNAL_API_KEY · DB_PASS — the app refuses to start if any is missing or weak.
  • Masked logs: client_id & tokens masked; the Client Secret and full access token are never stored.

🛡Encryption & surface

  • Encryption: in transit (Cloudflare Full strict + HTTPS) and at rest (MySQL 8 encryption-at-rest ON).
  • Loopback-only: uvicorn (127.0.0.1:8001), Redis & MySQL bind to localhost — only nginx is public.
  • MySQL 8: caching_sha2_password + FIPS handshake · InnoDB · DB_PASS required.
Backend · Batch updates

Scheduled jobs & how we run them live

Orchestratorpriority hourly · others daily

Refreshes USCIS case statuses on schedule; enqueues work for the executor by priority.

Executor≤ TPS · auto budget

Paces every USCIS call to the rate cap (5/s sandbox · 10/s prod); auto-regulates the daily budget.

Notifierevery 2 min

On a status change, sends Email · WhatsApp · Telegram; logged in notifications_log.

In the demo

  • Run the orchestrator manually (cron outside its normal schedule is the one exception USCIS allows).
  • Show a success run and an error response, with handling.
  • State the cadence: priority hourly, others daily; notifier every 2 min.
